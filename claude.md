## 🏗️ Architecture Requirements: GUI Tool Compatibility

**Critical Design Pattern:** All configurable parameters must be centrally accessible for dynamic GUI generation.

**Goal:** Enable a PyQt6 GUI tool to dynamically load any animation file and automatically create input fields for all adjustable parameters.

### Parameter Structure Requirements

**1. Central Parameter Dictionary**

All animation parameters MUST be defined in a central, easily parseable structure at the class level:

```python
class MyAnimation(Scene):
    """Animation demonstrating bond stretching."""

    # ✅ CORRECT: Central parameter dictionary
    PARAMETERS = {
        # Physical parameters
        "k": {
            "value": 500.0,
            "type": float,
            "unit": "eV/Å²",
            "description": "Spring constant for harmonic potential",
            "min": 0.0,
            "max": 2000.0
        },
        "r0": {
            "value": 1.5,
            "type": float,
            "unit": "Å",
            "description": "Equilibrium bond length",
            "min": 0.5,
            "max": 3.0
        },
        # Animation parameters
        "duration": {
            "value": 8.0,
            "type": float,
            "unit": "s",
            "description": "Total animation duration",
            "min": 1.0,
            "max": 30.0
        },
        "fps": {
            "value": 30,
            "type": int,
            "unit": "frames/s",
            "description": "Frames per second",
            "min": 10,
            "max": 60
        }
    }

    def construct(self):
        # Access parameters from central dictionary
        k = self.PARAMETERS["k"]["value"]
        r0 = self.PARAMETERS["r0"]["value"]
        # ... rest of implementation
```

**2. Parameter Dictionary Format**

Each parameter entry should include:
- `value`: The default value
- `type`: Python type (float, int, str, bool)
- `unit`: Physical unit (if applicable)
- `description`: Human-readable description
- `min`: Minimum allowed value (optional, for numeric types)
- `max`: Maximum allowed value (optional, for numeric types)

**3. What NOT to Do**

❌ **WRONG: Scattered hardcoded values**
```python
class BadAnimation(Scene):
    def construct(self):
        k = 500.0  # Hidden parameter, GUI can't find it
        r0 = 1.5   # Another hidden parameter
        duration = 8.0  # No metadata
```

❌ **WRONG: Instance variables without metadata**
```python
class BadAnimation(Scene):
    def setup_parameters(self):
        self.k = 500.0  # GUI can't extract this automatically
        self.r0 = 1.5
```

### Benefits of This Pattern

1. **GUI Auto-Generation:** PyQt6 tool can read PARAMETERS and create appropriate input widgets
2. **Documentation:** Parameters are self-documenting with descriptions and units
3. **Validation:** Min/max values enable automatic input validation
4. **Discoverability:** All parameters in one place, easy to find and modify
5. **Type Safety:** Explicit type information prevents errors

            "min": 0.5,
            "max": 3.0
        },
        "amplitude": {
            "value": 0.3,
            "type": float,
            "unit": "Å",
            "description": "Oscillation amplitude",
            "min": 0.1,
            "max": 1.0
        },
        "frequency": {
            "value": 0.5,
            "type": float,
            "unit": "Hz",
            "description": "Oscillation frequency",
            "min": 0.1,
            "max": 2.0
        },
        "duration": {
            "value": 8.0,
            "type": float,
            "unit": "s",
            "description": "Animation duration",
            "min": 1.0,
            "max": 30.0
        },
        "fps": {
            "value": 30,
            "type": int,
            "unit": "frames/s",
            "description": "Frames per second",
            "min": 10,
            "max": 60
        }
    }

    def construct(self):
        self.setup_scene()
        self.run_animation()

    def setup_scene(self):
        """Setup visualization elements."""
        # Extract parameters
        self.k = self.PARAMETERS["k"]["value"]
        self.r0 = self.PARAMETERS["r0"]["value"]
        # ... rest of setup

        duration = self.PARAMETERS["duration"]["value"]

### GUI Tool Integration

The PyQt6 tool will:
1. Import the animation file
2. Inspect the `PARAMETERS` class variable
3. Generate appropriate Qt widgets:
   - `QDoubleSpinBox` for float parameters
   - `QSpinBox` for int parameters
   - `QLineEdit` for string parameters
   - `QCheckBox` for bool parameters
4. Set min/max ranges from metadata
5. Display units and descriptions as labels/tooltips
6. Allow user to modify values before running animation

**This architecture ensures that animations remain standalone (can be run directly with ManimGL) while also being fully compatible with GUI-based parameter exploration.**

## 🔧 ManimGL Syntax (Essential)

**Import:**
```python
from manimlib import *
import numpy as np
```

**Key Commands:**
- `ShowCreation()` NOT `Create()`
- `axes.get_graph()` NOT `axes.plot()`
- `axes.coords_to_point()` NOT `axes.c2p()`
- `--write_file` NOT `--write_to_movie`

**Reference Lines:**
- Use `Line(axes.coords_to_point(x1,y), axes.coords_to_point(x2,y))` NOT `axes.get_horizontal_line()`
- Use `axes.coords_to_point()` for all coordinate transformations

**LaTeX in Tex():**
- ❌ NEVER use Unicode characters in `Tex()` or `Text()` if used in math context: `"μ"`, `"r₀"`, `"θ"` → LaTeX Error!
- ❌ NEVER use Unicode in `Text()` that might be confused with LaTeX: `"Schwerpunkt μ"` → Error!
- ✅ ALWAYS use LaTeX syntax: `r"r_0"` or `r"\theta_0"` or `r"\mu"`
- ✅ Use raw strings (`r"..."`) for LaTeX formulas
- ✅ For Greek letters in text: Either use `Tex(r"...\mu...")` or avoid Unicode entirely
- Examples:
  - ❌ `Tex("V = ½k(r-r₀)²")`
  - ✅ `Tex(r"V = \frac{1}{2}k(r-r_0)^2")`
  - ❌ `Text("Schwerpunkt μ")`
  - ✅ `Text("Schwerpunkt")` or `Tex(r"\text{Schwerpunkt } \mu")`

**ParametricSurface Styling:**
- ❌ NO `set_style()` method exists!
- ❌ NO `set_fill()` method exists!
- ✅ Use `surface.set_color(COLOR)` and `surface.set_opacity(value)`
- Example:
  ```python
  surface = ParametricSurface(func, ...)
  surface.set_color(BLUE)
  surface.set_opacity(0.3)
  ```

**Animation Performance:**
- ❌ NEVER `remove()` and `add()` entire groups in animation loops
- ✅ Only remove/add objects that actually change (e.g., arcs, polygons)
- ✅ Use `put_start_and_end_on()` for lines, `move_to()` for atoms
- Bad: `self.remove(group); group = VGroup(...); self.add(group)` in loop
- Good: Only update positions of existing objects

## 📐 Best Practices for ManimGL

### 1. Basic Structure of a Scene

Every animation must be contained in a class that inherits from `Scene`. The main logic is in the `construct` method.

```python
from manimlib import *

class MyAnimationScene(Scene):
    def construct(self):
        # 1. Create objects (MObjects)
        circle = Circle()
        text = Text("Hello, Manim!")

        # 2. Add objects to the scene
        self.add(circle)

        # 3. Play animations
        self.play(Write(text))

        # 4. Wait for a while
        self.wait(2)
```

**Important Commands:**
- `self.add(mobject)`: Adds an object immediately without animation
- `self.play(animation, ...)`: Plays one or more animations (e.g., `Write`, `FadeIn`, `ShowCreation`)
- `self.wait(duration)`: Pauses the animation for a specified duration in seconds

**Executing a Scene:**
```bash
manimgl my_script.py MyAnimationScene
```

### 2. Loop-Based Animation

For dynamic simulations (e.g., molecular dynamics, Monte Carlo), a `for` loop is the preferred approach. This is simpler and more direct than using `add_updater`.

**Key Concept:** Update MObject properties within the loop and then call `self.wait(dt)` to redraw the scene.

```python
# From bond_stretching.py
class BondStretching(Scene):
    def construct(self):
        # ... Setup ...
        self.create_molecule()
        self.create_potential_curve()

        # Animation loop
        duration = 8.0
        fps = 30
        frames = int(duration * fps)

        for frame in range(frames):
            t = frame / fps

            # 1. Calculate current state
            r_current = self.r0 + self.amplitude * np.sin(2 * np.pi * self.frequency * t)

            # 2. Update MObjects (NO .animate syntax here!)
            self.update_molecule_positions(r_current)

            energy = self.harmonic_potential(r_current)
            point_on_curve = self.axes.coords_to_point(r_current, energy)
            self.current_dot.move_to(point_on_curve)

            # 3. Wait for one frame to make changes visible
            self.wait(1/fps)
```

### 3. Creating 3D Animations (`ThreeDScene`)

For 3D animations, special classes and methods must be used.

#### a. `ThreeDScene` as Base Class
Your scene class must inherit from `ThreeDScene`.

```python
from manimlib import *

class My3DScene(ThreeDScene):
    def construct(self):
        # ... 3D logic here ...
```

#### b. Camera Control
The camera is controlled differently than in older Manim versions.

- **Correct Method:** `self.camera.frame.set_euler_angles(theta=..., phi=...)`
- **Wrong Method:** `self.set_camera_orientation(...)` leads to an `AttributeError`

```python
# Correct camera setup from torsion_angle_3d.py
class TorsionAngle3D(ThreeDScene):
    def setup_camera(self):
        """Setup 3D camera view"""
        self.camera.frame.set_euler_angles(theta=-45*DEGREES, phi=70*DEGREES)
```

#### c. Mixing 3D Objects and 2D Overlays (HUD)
A common use case is displaying 3D objects that rotate while 2D elements (text, graphs) remain static in the foreground.

- **3D Objects:** Create `Sphere`, `Cube` or position 2D objects with 3D vectors (e.g., `Sphere().move_to([1, 2, 3])`)
- **2D Overlays:** Create normal 2D objects (`Text`, `Axes`, `VGroup`)
- **IMPORTANT:** Call the `.fix_in_frame()` method for each 2D overlay object

```python
# Example from torsion_angle_3d.py
class TorsionAngle3D(ThreeDScene):
    def construct(self):
        # ...
        # 2D title as overlay
        title = Text("Torsion Angle").scale(0.6)
        title.to_edge(UP)
        title.fix_in_frame() # <-- This call is crucial!
        self.add(title)

        # 3D object
        sphere = Sphere(radius=1)
        self.add(sphere)

        # Camera rotation
        self.begin_ambient_camera_rotation(rate=0.1)
        self.wait(5)
```

#### d. `Group` vs. `VGroup` in 3D
In 3D scenes, it is **mandatory** to use `Group` instead of `VGroup` to group 3D objects (like `Sphere`) or a mix of 3D and 2D objects. `VGroup` is only for 2D vector-based objects (`VMobject`) and otherwise leads to an `Exception`.

### 4. Advanced Best Practices

#### a. Structuring Complex Scenes
Split the `construct` method into logical private methods. This significantly improves readability and maintainability.

```python
# Good example from forcefield_visualization.py
class ForceFieldVisualization(Scene):
    def construct(self):
        self.setup_forcefield_parameters()
        self.setup_layout()
        self.create_molecular_system()
        self.run_forcefield_demonstration()
```

#### b. Handling Text and Formulas
- **Normal Text:** Use `Text("My Text")`
- **Mathematical Formulas (LaTeX):** Use `Tex(r"\frac{a}{b}")`
  - **Important:** Always use raw strings (`r"..."`), to avoid problems with backslashes
  - `manimgl` requires a working LaTeX installation

#### c. Data Visualization with `Axes`
- Create axes with `Axes(...)`
- **Important:** Always convert data coordinates to screen coordinates with `self.axes.coords_to_point(x_val, y_val)`
- To plot functions, use `self.axes.get_graph(my_function, x_range=[...])`

#### d. Performance with Many Objects
When creating thousands of objects (e.g., points in a Monte Carlo simulation), limit the number of simultaneously displayed objects to maintain performance.

```python
# Good example from monte_carlo_pi.py
self.displayed_points = []
self.max_displayed_points = 500

# In the animation loop:
visual_point = Dot(...)
self.displayed_points.append(visual_point)

if len(self.displayed_points) > self.max_displayed_points:
    old_point = self.displayed_points.pop(0)
    self.remove(old_point) # Removes the oldest object from the scene

self.add(visual_point)
```

#### e. Internationalization (i18n)
For multilingual animations, using a `STRINGS` dictionary is a robust pattern.

```python
# Pattern from your scripts
LANGUAGE = "DE"  # or "EN"

STRINGS = {
    "DE": {"title": "Mein Titel"},
    "EN": {"title": "My Title"}
}

def get_string(key):
    return STRINGS[LANGUAGE][key]

# Usage:
title = Text(get_string("title"))
```

### 5. Common Errors and How to Avoid Them

1. **Error: Object doesn't appear.**
   - **Cause:** The MObject was created but never added to the scene with `self.add(mobject)` or `self.play(Animation(mobject))`
   - **Solution:** Make sure every object that should be visible is added

2. **Error: Animations don't play; everything appears immediately at the end.**
   - **Cause:** Object properties were changed in a loop, but `self.wait(dt)` was forgotten inside the loop. Without `wait` or `play`, Manim executes the code immediately and shows only the final result
   - **Solution:** Add `self.wait(dt)` at the end of each iteration of an animation loop

3. **Error: `AttributeError: 'NoneType' object has no attribute 'copy'` when animating.**
   - **Cause:** Incorrect use of `self.play()`. `my_object.move_to(X)` returns `None`
   - **Wrong:** `self.play(my_object.move_to(X))`
   - **Correct:** `self.play(my_object.animate.move_to(X))`
   - **Note:** In the shown scripts, the `.animate` syntax is rarely needed because animation is controlled by the loop and `self.wait()`. It's mainly useful for simple, one-time transitions

4. **Error: Animation becomes extremely slow or crashes.**
   - **Cause:** Thousands of MObjects are added to the scene without removing old ones
   - **Solution:** Implement a limit for displayed objects, as described in section 4.d

5. **Error: `AttributeError: 'MyScene' object has no attribute 'my_label'`**
   - **Cause:** Wrong initialization order. An update method accesses a `self` attribute that hasn't been created yet
   - **Example Problem:**
     ```python
     # WRONG
     def create_molecule(self):
         # ...
         self.update_molecule(0) # ERROR: self.angle_label doesn't exist here yet
         # ...
         self.angle_label = DecimalNumber(0)
         self.add(self.angle_label)

     def update_molecule(self, angle):
         self.angle_label.set_value(angle) # <-- Leads to AttributeError
     ```
   - **Solution:** Ensure all MObjects needed by update methods are initialized **before** the first call to these methods
     ```python
     # CORRECT
     def create_molecule(self):
         # ...
         # Create label FIRST
         self.angle_label = DecimalNumber(0)
         self.add(self.angle_label)
         # ...
         # THEN call the update method
         self.update_molecule(0)

     def update_molecule(self, angle):
         self.angle_label.set_value(angle)
     ```

6. **Error: `Exception: Only VMobjects can be passed into VGroup`**
   - **Cause:** Attempted to insert a non-vectorized object (like a 3D `Sphere`) into a `VGroup`
   - **Wrong:** `my_group = VGroup(Sphere(), Circle())`
   - **Solution:** Use the generic `Group` class when grouping 3D objects or a mix of different object types
   - **Correct:** `my_group = Group(Sphere(), Circle())`

## 📊 4-Quadrants Layout Standard

Standard layout for molecular dynamics animations:

```
┌─────────────────────────┬─────────────┐
│                         │  Energy     │
│   Main Visualization    │  vs Time    │
│   (Molecule/3D)         ├─────────────┤
│                         │  Parameter  │
│                         │  vs Time    │
└─────────────────────────┴─────────────┘
```

**Layout Principles:**
- **Left**: Main visualization (large)
- **Right top**: Energy vs Time
- **Right bottom**: Parameter vs Time
- **Font sizes**: 8-14px
- **Spacing**: `buff=0.2` minimum
- Reaction/molecule box in left panel (2 quadrants), energy and time tracking in right diagrams
